<task type="auto">
  <name>Task 1: Fix Model Selection Persistence</name>
  <files>ui/app.js</files>
  <action>
    Refactor `fetchModels()` in `ui/app.js` to restore the selected model from `providerSettingsCache` instead of the `#providerModelSelect` DOM element.
    
    Currently, `updateProviderUI()` clears `providerModelSelect.innerHTML` before `fetchModels()` is called. `fetchModels()` then tries to read `providerModel.value`, which is empty.
    
    Fix:
    - Pass the cached model value to `fetchModels()` or access it via `providerSettingsCache[provider].model`.
    - After populating the dropdown, set `providerModelSelect.value` to the cached value if it exists in the fetched list.
  </action>
  <verify>
    <automated>grep -A 20 "async function fetchModels" ui/app.js | grep "providerSettingsCache"</automated>
  </verify>
  <done>Selected model is correctly restored after async model fetch completes.</done>
</task>

<task type="auto">
  <name>Task 2: Switch to Session Storage for Tab Isolation</name>
  <files>ui/app.js</files>
  <action>
    Replace all instances of `localStorage` usage for `activeSessionId` with `sessionStorage`.
    
    This includes:
    - `localStorage.getItem('activeSessionId')`
    - `localStorage.setItem('activeSessionId', ...)`
    - `localStorage.removeItem('activeSessionId')`
    
    Keep `localStorage` for the 'theme' setting as that should be global across tabs.
  </action>
  <verify>
    <automated>! grep "localStorage.*activeSessionId" ui/app.js && grep "sessionStorage.*activeSessionId" ui/app.js</automated>
  </verify>
  <done>Active session ID is now stored in sessionStorage, enabling independent runs in multiple tabs.</done>
</task>
