# Phase 2 Plan: Magi-Optics Telemetry & Logging Completion

## Objective
Fix broken observability integrations (Loki/Bifrost) and complete the missing high-fidelity terminal UI components as per the "Magi-Optics Telemetry Overhaul" spec, without duplicating already implemented work.

## Key Files & Context
- `pipeline/telemetry.py`: `TelemetryEmitter` currently uses incorrect hardcoded Loki URL.
- `pipeline/observability.py`: Central logging config, needs endpoint consolidation.
- `pipeline/orchestrator.py`: Needs `rich.progress` for extraction loop and consolidated run summary.
- `pipeline/llm_client.py`: Already has some `console.status` and `llm.call` emits, needs `stop_event` integration.
- `pageindex/page_index.py`: Needs logger safety and `stop_event` propagation.

## Implementation Steps

### Step 2.1: Fix Broken Observability (Empty Dashboards)
- [ ] **Task 2.1.1: Align Telemetry Endpoints**
  - Refactor `pipeline/telemetry.py` to use `SYSTEM_LOKI_URL` and `SYSTEM_BIFROST_TOKEN` from `observability.py`.
  - Remove hardcoded `BACKBONE_LOKI_URL` and obfuscated tokens from `telemetry.py`.
- [ ] **Task 2.1.2: Standardize Event Payloads**
  - Ensure `TelemetryEmitter` formats payloads correctly for the unified Loki/Bifrost gateway.
  - Verify `llm.call`, `run.started`, and `step.completed` events are reaching the dashboard.

### Step 2.2: Complete High-Fidelity Terminal UI
- [ ] **Task 2.2.1: Implement Extraction Progress Bar**
  - Integrate `rich.progress.Progress` into `PipelineOrchestrator.run()` to replace messy logs during the node extraction loop.
  - Show per-node status and overall completion percentage.
- [ ] **Task 2.2.2: Consolidated Run Summary (PIPE-01)**
  - Implement `PipelineOrchestrator._print_run_summary()` to print a clean, formatted box with Run ID, Provider, Model, and Mode exactly once.
  - Clean up redundant `logger.info` calls in `orchestrator.py` and `main.py`.
- [ ] **Task 2.2.3: Animated Spinners for PageIndex**
  - Ensure `console.status()` is used during the PageIndex tree building phase to provide visual feedback.

### Step 2.3: Reliable Cancellation & Logger Safety
- [ ] **Task 2.3.1: Global stop_event Integration (PIPE-02)**
  - Update `LLMClient.complete()` and `complete_json()` to check `self.stop_event.is_set()` before each attempt.
  - Ensure all agent loops in `pipeline/agents/` check the `stop_event` and exit gracefully.
- [ ] **Task 2.3.2: PageIndex Logger & Stop Safety (PIPE-03)**
  - Patch `pageindex/` modules to safely handle `logger=None`.
  - Propagate `stop_event` into `pageindex` tree building loops.
- [ ] **Task 2.3.3: Automated Handler Cleanup**
  - Implement a `run_logger` context manager in `observability.py`.
  - Refactor `ui/server.py` to use this context manager for reliable resource cleanup.

## Verification & Testing
- **Observability:** Confirm logs and events are visible in Grafana/Loki via the unified gateway.
- **Terminal UI:** Verify the extraction progress bar and animated spinners appear correctly in the console.
- **Cancellation:** Test cancellation from the UI and verify immediate pipeline stoppage.
- **Safety:** Verify `PageIndex` runs without error when `logger` is not provided.
