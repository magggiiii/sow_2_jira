# Argus Opt-Out Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a simple "turn off button" for Argus remote telemetry, allowing users to opt-out during installation and keep all logs exclusively local.

**Architecture:** Introduce an `ARGUS_SYNC_ENABLED` environment variable. When false, bypass OpenLLMetry initialization and OTel metric recording in `pipeline/observability.py` and `pipeline/llm_client.py`.

**Tech Stack:** Python, Bash.

---

### Task 1: Update Application Instrumentation (Opt-Out Logic)

**Files:**
- Modify: `pipeline/observability.py`
- Modify: `pipeline/llm_client.py`

- [ ] **Step 1: Read `ARGUS_SYNC_ENABLED` in `pipeline/observability.py`.**
Determine if synchronization should be enabled based on the environment variable. Default to `false` if not set.

- [ ] **Step 2: Conditionally initialize `init_argus`.**
Wrap the call to `init_argus()` in an `if ARGUS_SYNC_ENABLED:` block.

- [ ] **Step 3: Provide safe fallbacks for OTel features.**
Ensure `otel_log_format` and `run_log_format` handle the absence of a valid OTel span context gracefully (e.g., returning "0" * 32 for the trace ID).
Ensure the `@trace_span` decorator doesn't crash or attempt to record spans if OTel is disabled. If disabled, it can simply execute the wrapped function.

- [ ] **Step 4: Conditionally record metrics in `pipeline/llm_client.py`.**
Import `ARGUS_SYNC_ENABLED` from `observability.py`.
Wrap the calls to `llm_token_usage.add` and `llm_operation_duration.record` in an `if ARGUS_SYNC_ENABLED:` block to prevent errors when instruments aren't initialized.

- [ ] **Step 5: Commit changes.**
