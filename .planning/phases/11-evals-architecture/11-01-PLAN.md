  </read_first>
  <action>
    1. Update `config/admin/bifrost.admin.yaml` to include a local provider entry for `ollama` (pointing to `http://host.docker.internal:11434` or similar) to support local evaluation models.
    2. Define LLM-as-a-judge prompt templates in `infra/admin/evaluator/prompts.py`. The prompt must ask the judge to:
       - Compare the list of `generated_tasks` against `ground_truth_tasks` for a given `source_text`.
       - Identify semantically matching tasks (even if titles differ).
       - Score **Alignment** (0-1): Does the generated set capture the Epic's primary goal?
       - Score **Recall** (0-1): What % of ground-truth requirements are represented in the generated tasks?
       - Score **Hallucination** (0-1): Are there tasks that describe work not present in the SOW?
    3. Implement `HierarchicalEvaluator` class in `infra/admin/evaluator/logic.py`.
       - Use LangChain `ChatOpenAI` configured with `openai_api_base="http://s2j-admin-bifrost:8081/v1"` and a virtual key from `bifrost.admin.yaml`.
       - Method `evaluate(source_text, generated_tasks, ground_truth_tasks)` returns a dict of scores and reasoning.
  </action>
  <verify>
    <automated>grep -q "HierarchicalEvaluator" infra/admin/evaluator/logic.py && grep -q "ollama" config/admin/bifrost.admin.yaml</automated>
  </verify>
  <acceptance_criteria>
    - `HierarchicalEvaluator` class exists and accepts List[Dict] for tasks.
    - `bifrost.admin.yaml` contains `ollama` provider.
    - `prompts.py` contains specific logic for EPIC -> ticket hierarchy evaluation.
  </acceptance_criteria>
  <done>Core evaluation logic and local model support are implemented.</done>
