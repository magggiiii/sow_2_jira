# Docker Distribution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a one-command `curl | bash` installer that sets up SOW-to-Jira using pre-built images and an interactive credential wizard.

**Architecture:** A unified `install.sh` script detects OS, ensures Docker is running, downloads distribution artifacts (`docker-compose.yml`, `config/tempo.yaml`), runs a credential wizard, and sets up a persistent `sjt` alias.

**Tech Stack:** Bash, Docker Compose, LiteLLM (via pre-built image), Jira API.

---

### Task 1: Create Distribution-Ready Docker Compose

**Files:**
- Create: `docker-compose.dist.yml`

- [ ] **Step 1: Create `docker-compose.dist.yml` from the current `docker-compose.yml` but use pre-built images.**

```yaml
services:
  app:
    image: calib.dev/mageswaran/sow-to-jira:v1.0
    container_name: sow-to-jira-app
    init: true
    ports:
      - "127.0.0.1:8000:8000"
    volumes:
      - ./data:/app/data
      - ./config:/app/config:ro
    environment:
      - SOW_DATA_DIR=/app/data
      - PYTHONUNBUFFERED=1
      - BIFROST_GATEWAY_URL=${BIFROST_GATEWAY_URL}
      - BIFROST_BACKBONE_TOKEN=${BIFROST_BACKBONE_TOKEN}
    networks:
      - sow-internal
    restart: unless-stopped

  # ... other services (bifrost, loki, tempo, grafana) from current docker-compose.yml
```

- [ ] **Step 2: Commit `docker-compose.dist.yml`.**

```bash
git add docker-compose.dist.yml
git commit -m "feat(dist): add distribution-ready docker-compose manifest"
```
